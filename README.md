## Explicacion de cada caracter y categoría de la lista.<br>

```yaml
Caracteres_disponibles:
	   ========================
	   🧩 Carácteres de sintaxis HTML
	   ========================
	  #1: "<"   → abre etiqueta HTML (permite inyectar nuevas etiquetas)
	  #2: ">"   → cierra etiqueta HTML (finaliza etiquetas abiertas)
	  #3: "/"   → cierre de etiqueta (</tag>, autocierre <br/>)
	  #4: "="   → asignación de atributos (<img src=x>)
	  #5: "\""  → delimitador de atributos (puede romper y salir del atributo)
	  #6: "'"   → delimitador de atributos (igual que comillas dobles)
	  #7: "`"   → usado en atributos inline y JS (permite template literals)
	  #8: "!"   → usado en <!DOCTYPE> o <!-- comentarios --> (control del parser)
	  #9: "?"   → usado en <?php ?>, ? en URLs (querystring)
	  #10: "#"  → marcador de fragmento en URLs (<a href="#frag">)
	   =========================
	   💻 Carácteres de sintaxis JavaScript
	   =========================
	  #11: "{"   → inicio de bloque de código
	  #12: "}"   → fin de bloque de código
	  #13: "("   → llamada a función o agrupación de expresión
	  #14: ")"   → cierre de llamada de función
	  #15: "["   → acceso a arrays/objetos (obj['x'])
	  #16: "]"   → cierre de acceso a arrays/objetos
	  #17: ";"   → separador de sentencias
	  #18: ","   → separador de parámetros o expresiones
	  #19: ":"   → asignación de propiedades en objetos
	  #20: "+"   → concatenación o suma
	  #21: "-"   → resta o negación
	  #22: "*"   → multiplicación
	  #23: "/"   → división o inicio de regex
	  #24: "%"   → operador módulo
	  #25: "."   → acceso a propiedades (obj.prop)
	  #26: "|"   → OR bitwise o logical pipe
	  #27: "&"   → AND bitwise
	  #28: "^"   → XOR bitwise
	  #29: "~"   → NOT bitwise
	  #30: "?"   → operador ternario (condición ? A : B)
	  #31: ":"   → parte del operador ternario
	  #32: "\\"  → escape de caracteres
	  #33: "$"   → válido en nombres de variables/funciones ($evil())
	   =========================
	   🌐 Carácteres especiales de URL/HTTP
	   =========================
	  #34: "%"   → inicio de secuencias %XX (URL encoding, evasión de filtros)
	  #35: "&"   → separador de parámetros (?a=1&b=2)
	  #36: "?"   → inicio de querystring (/path?x=1)
	  #37: "#"   → marcador de fragmento (/index#x)
	  #38: "+"   → representa espacio (x-www-form-urlencoded)
	  #39: ":"   → separador en esquemas (http:)
	  #40: "@"   → usado en credenciales de URL (user:pass@host)
	   =========================
	   🧙 Carácteres escapables en HTML/JS
	   =========================
	  #41: "&#NN;"   → entidades decimales HTML (&#60; = <)
	  #42: "&#xNN;"  → entidades hexadecimales HTML (&#x3c; = <)
	  #43: "\\xNN"   → escape hexadecimal JS (\x3c = <)
	  #44: "\\uNNNN" → escape unicode JS (\u003c = <)
	  #45: "\\NNN"   → escape octal JS (\074 = <) (no siempre soportado)
	  #46: "&lt;"    → entidad HTML (<)
	  #47: "&gt;"    → entidad HTML (>)
	  #48: "&quot;"  → entidad HTML (")
	  #49: "&apos;"  → entidad HTML (')
	  #50: "&amp;"   → entidad HTML (&)
	   =========================
	   🎭 Carácteres invisibles / raros
	   =========================
	  #51: "\u200B" → zero-width space (no visible, rompe validaciones)
	  #52: "\u200C" → zero-width non-joiner
	  #53: "\u200D" → zero-width joiner
	  #54: "\u2060" → word joiner
	  #55: "\uFEFF" → BOM (Byte Order Mark)
	  #56: "\u180E" → Mongolian vowel separator (obsoleto pero soportado)
	  #57: "\u00A0" → espacio no separable (non-breaking space)
	  #58: "\u202E" → BIDI override (invierte el orden visual del texto)
	   =========================
	   💣 Carácteres de ruptura contextual
	   =========================
	  #59: "\">"    → cierra atributo y abre nueva etiqueta (<img src=x" onerror=...>)
	  #60: "</"     → inicia cierre de etiqueta (</script>)
	  #61: "--!>"   → termina comentario HTML (<!-- X --!>)
	  #62: "-->"    → fin de comentario HTML
	  #63: "]]>"    → cierra bloques CDATA
	  #64: "/**/"   → comentario CSS/JS para insertar sin romper sintaxis
	  #65: "//"     → comentario de línea JS (rompe resto del código)
	  #66: "<!--"   → inicia comentario HTML
	   =========================
	   ⚡ Bonus: separadores multi-contexto
	   =========================
	  #67: ","   → separador de expresiones (ej: (1,alert(1)))
	  #68: "|"   → pipe lógico / filtrado en plantillas
	  #69: "~"   → operador bitwise NO (~alert.length)
	  #70: "^"   → XOR, rompe lógicas raras
	  #71: "¬"   → símbolo lógico NO (no estándar pero aceptado en algunos parsers)
```
